FUNDAMENTAL COUNTING PRINCIPLE
AND TREE DIAGRAMS

One method for determining the number of possible outcomes for an event followed by another event in a particular sample space is using the Fundamental Counting Principle. This property states that if a particular event A occurs in a ways and is followed by an event B that occurs in b ways, then the event A followed by event B occurs in ab ways.

For example, if you have two pairs of different colored pants (tan and charcoal) and three different colored shirts (white, blue, and green), how many different outfit variations could you wear?

Since there are two choices in pant selections and three choices in shirt selections, there are 23 outfit variations available to you. Therefore, there are 6 different outfits (or arrangements) that result from these options.

The Fundamental Counting Principle easily helps us determine how many ways there are to make arrangements, but it does not help us to actually see and list the actual outcomes in the sample space. As a result, you may want to make a tree diagram. Additionally, tree diagrams can show probabilities that are based on some other event happening first. We call these conditional probabilities.

Arrangements

Now that you have mastered determining possible outcomes, we introduce the arrangements in a particular order. An arrangement or list in which the order is important is a permutation. For example, in a class of 25 students, if a president, vice-president and secretary are chosen, the order in which these are chosen matters. Choosing Amy for president, Bill for vice-president and Charlie for secretary (A-B-C) is different than choosing Bill for president, Charlie for vice-president and Amy for secretary (B-C-A). A-B-C is a different permutation than B-C-A.

On the other hand, if the order is not important, it is called a combination. For example, in a class of 25 students, a committee of three students is chosen, the order in which these are chosen does NOT matter.
